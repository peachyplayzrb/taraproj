# CS Paper Search Engine

A method-aware academic search engine over arXiv Computer Science papers, built as a course assignment. The system combines TF-IDF retrieval with Part-of-Speech (POS) weighting and click-through re-ranking to demonstrate transparent, controllable, and observable ranking.

---

## What It Does

Given a text query (e.g. *"graph neural network node classification"*), the engine:

1. Preprocesses the query through the same NLP pipeline used to index documents (lowercase → punctuation removal → tokenization → stopword removal → stemming)
2. Vectorizes the query using a pre-fitted TF-IDF model (50k features, 1-2grams)
3. Applies POS weights (nouns × 2.0, verbs × 1.2, hyphenated compounds × 1.8, other × 0.5)
4. Computes cosine similarity against ~50k–500k indexed documents
5. Blends with click-through history for personalized re-ranking
6. Returns ranked results with all scoring dimensions visible

## Architecture

```
arXiv JSON snapshot
       │
       ▼
Untitled4 (8).ipynb  ← Run in Google Colab
  ├── Filter CS papers, 2015+, abstract ≥ 200 chars
  ├── Combine title (×2) + abstract
  ├── NLP pipeline: lowercase → tokenize → stopwords → stem
  ├── POS tag → build per-term weight vector
  ├── TF-IDF vectorizer (50k features, bigrams)
  ├── Multiply columns by POS weights → tfidf_pos_matrix
  └── Serialize all artifacts to Google Drive
       │
       ▼ (copy .pkl / .npz files locally)
       │
app.py  ← Flask backend (python app.py)
  ├── Load vectorizer, matrices, doc metadata, click_store
  ├── GET /search?q=...&sort=...&k=10  →  ranked JSON
  └── POST /click                       →  persist click event
       │
       ▼
templates/index.html  ← Single-page UI
  ├── Search bar + sort dropdown (6 strategies) + top-k selector
  ├── Result cards with score pills (blended / cosine / click / popularity / year)
  └── Click recording → updates click_store for future queries
```

## Dataset Setup

The dataset is the **arXiv metadata snapshot** maintained by Cornell University.

1. Download from Kaggle: [Cornell-University/arxiv](https://www.kaggle.com/datasets/Cornell-University/arxiv)
2. The file is `arxiv-metadata-oai-snapshot.json` (~4 GB, JSONL format)
3. Upload it to your Google Drive at:
   ```
   /content/drive/MyDrive/arxiv-metadata-oai-snapshot.json
   ```

> **Note:** The `.npz` matrix files in this repo are pre-built from the dataset so the Flask app can run without re-executing the notebook, as long as the other `.pkl` files are also present.

1. Open [Google Colab](https://colab.research.google.com/) and upload `Untitled4 (8).ipynb`
2. Mount Google Drive when prompted (the notebook loads data from and saves artifacts to Drive)
3. Ensure the arXiv snapshot is in place (see Dataset Setup above)
4. Run all cells in order — the pipeline is:

| Phase | What happens |
|---|---|
| Cell 1 | Install/import libraries, download NLTK data |
| Cells 2–5 | Load arXiv JSON in chunks, filter to CS papers |
| Cells 6–13 | Filter abstracts ≥ 200 chars, filter to 2015+, explore distributions |

| Cell 29 | Build 50k-feature TF-IDF matrix |
| Cells 30–32 | Build POS weight vector, multiply into TF-IDF matrix |

## Ranking Strategies

The Flask app exposes 6 sort modes selectable from the UI dropdown:

| Mode | Formula | Use case |
|---|---|---|
| `blended` (default) | 70% cosine + 30% query-specific clicks | Balances relevance and personal engagement |
| `relevance` | Pure cosine similarity (POS-weighted) | Strict keyword relevance |
| `trending` | 60% cosine + 40% global click popularity | Surfaces frequently clicked results |
| `popularity` | Sorted by total clicks across all queries | Most-clicked papers overall |
| `newest` | Cosine threshold, then sort by year descending | Most recent relevant papers |
| `oldest` | Cosine threshold, then sort by year ascending | Foundational/earliest relevant papers |

## NLP Pipeline

Both documents and queries go through the same pipeline so representations are compatible:

```
raw text
  → lowercase
  → remove punctuation (keep hyphens: "self-attention")
  → remove digits
  → word_tokenize (NLTK)
  → remove stopwords
  → PorterStemmer
  → join tokens → TF-IDF vector
  → multiply by pos_weight_diag
```

POS weights applied at index-build time:
- Nouns (`NN*`) → **2.0×**
- Hyphenated compounds → **1.8×**
- Verbs (`VB*`) → **1.2×**
- Adjectives (`JJ*`) → **1.0×**
- Everything else → **0.5×**

---

## Click-Through Re-ranking

Clicks are stored as `{normalized_query → {doc_id → click_count}}`. Query normalization sorts stemmed tokens so *"deep learning"* and *"learning deep"* map to the same key.

The blended score for a document `d` given query `q` is:

$$\text{score}(d,q) = \alpha \cdot \text{cosine}(d,q) + (1-\alpha) \cdot \text{click\_score}(d,q)$$

Default $\alpha = 0.7$ keeps retrieval quality dominant while allowing popular results to surface.
